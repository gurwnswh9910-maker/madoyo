# 📂 프로젝트 구조 및 사용 설명서 (Madoyo System)

이 문서는 프로젝트의 폴더 구조 원칙과 `작동중코드` 내 주요 파일들의 사용법을 정의합니다.

---

## 1. 폴더 구조 원칙 (Directory Structure)

### ✅ 작동중코드 (Active Code)
**"현재 실제로 서비스/업무에 사용되는 핵심 코드"**
- **카피 생성 (Copy Generation)**: MAB 알고리즘 기반 카피 생성 및 최적화.
- **크롤링 (Crawling)**: 스레드 데이터 수집.
- **임베딩 (Embedding)**: 데이터 벡터화 및 저장.
- **필수 종속성 (Dependencies)**: 위 기능들이 작동하는 데 필요한 모듈들.

### 🧪 연구소 (Research Lab)
**"분석, 실험, 테스트, 혹은 더 이상 쓰지 않는 코드"**
- **데이터 분석 (Analysis)**: 성과 분석, 통계 리포트 생성 (예: 요일별 성과 분석).
- **벤치마크 (Benchmark)**: 성능 테스트용 스크립트.
- **레거시/실험 (Legacy/Experiments)**: 개발 중이거나 보관용 코드.

## 2. 📘 작동중코드 (Active Code) 사용 설명서

`작동중코드` 폴더에 있는 주요 스크립트의 역할과 실행 방법입니다.

### 1️⃣ 카피 최적화 및 생성 (Copy Optimization)
**메인 실행 파일**: `optimize_copy_v2.py`
- **역할**: 사용자가 입력한 초안 카피를 바탕으로, MAB(Multi-Armed Bandit) 알고리즘과 고성과 성공 패턴을 적용하여 **최적화된 카피 3개**를 제안합니다.
- **주요 기능**:
    - **유사 성공 사례 검색**: 입력 카피와 비슷한데 "초대박"이 났던 과거 게시물을 찾아 벤치마킹합니다 (Embedding/Contrastive Prompting).
    - **전략 적용**: 'Hook 강화', '가성비 강조' 등 검증된 전략을 프롬프트에 주입하며, LLM이 최근 우수 사례들로부터 제품에 알맞은 '치트키 맥락(예: 엘리베이터 공간)'을 **동적으로 동기화**하여 제안합니다.
    - **성능 예측**: 생성된 카피가 얼마나 잘 될지(MSS 점수)를 임베딩 유사도와 **의미론적 붕괴 페널티(Semantic Penalty)** 공식으로 검증 후 Top 3만 보여줍니다.
- **실행 방법**:
  ```bash
  python 작동중코드/optimize_copy_v2.py
  ```

### 2️⃣ 데이터 수집 (Crawling)
**메인 실행 파일**: `profilecollect.py`
- **역할**: 스레드(Threads)에서 게시물 데이터를 수집합니다.
- **기능**:
    - **본문/조회수/좋아요** 등 핵심 지표 수집.
    - **첫 댓글 조회수** 수집 (MSS 지표 계산용).
    - **모드 지원**: 빠른 모드 vs 안전 모드(403 방지).
- **실행 방법**:
  ```bash
  python 작동중코드/profilecollect.py
  ```

### 3️⃣ 임베딩 및 데이터 피드백 (Embedding & Feedback)
**메인 실행 파일**: `data_feedback_loop_v2.py`
- **역할**: 수집된 데이터(`dotori.xlsx` 등)를 로드하여 **임베딩(벡터)**을 업데이트하고, MAB 엔진을 최신 상태로 학습시킵니다.
- **언제 실행하나요?**: 새로운 데이터를 엑셀에 추가했거나, `dotori.xlsx`를 업데이트한 후에 실행합니다.
- **실행 방법**:
  ```bash
  python 작동중코드/data_feedback_loop_v2.py
  ```

### 4️⃣ 기타 핵심 모듈 (Core Modules)
직접 실행하지 않지만, 위 프로그램들이 사용하는 부품들입니다.
- `mab_engine_v2.py`: MAB 알고리즘 엔진 (어떤 전략이 좋은지 계산).
- `strategy_clusterer.py`: 게시물 전략 분류기 (어떤 글이 어떤 전략인지 판단).
- `contrastive_prompter.py`: 대조 학습용 프롬프트 생성기 (잘 된 글 vs 망한 글 비교).
- `embedding_utils.py`: Gemini API를 통한 임베딩 처리 유틸리티.
- `copy_generator_v2.py`: 실제 LLM 프롬프트를 조립하는 생성기 (동적 치트키/상황 맥락 주입기능 포함).
- `copy_scorer_v3.py`: 생성된 카피 점수 매기기 (순수 로컬 임베딩 코사인 유사도 연산 + 타겟 제품 클러스터와의 3제곱 거리 페널티 문맥 검증).

## 3. 관리 가이드
- **파일 이동**: 코드를 수정하다가 새로운 기능을 만들었는데, 아직 테스트 중이라면 `연구소`에서 작업하세요.
- **정식 배포**: 테스트가 끝나고 실제 업무에 매일 쓰게 된다면 `작동중코드`로 옮기고, 이 문서를 업데이트하세요.
- **임베딩 업데이트**: 데이터가 쌓일수록 AI가 똑똑해집니다. 주기적으로 `data_feedback_loop_v2.py`를 돌려주세요.
