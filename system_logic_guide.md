# 스레드 바이럴 카피 최적화 엔진: 로직 및 작동 가이드

이 문서는 데이터 분석과 AI를 결합하여 고성과 SNS 카피를 생성하는 시스템의 내부 로직과 기술적 배경을 설명합니다.

---

## 🚀 1. 전체 프로세스 (Workflow)

시스템은 '데이터 분석 - 전략 도출 - 생성 - 검증 - 학습'의 5단계 순환 구조를 가집니다.

1.  **입력 (Input)**: 홍보할 제품 정보와 기본 카피를 입력합니다.
2.  **데이터 점수화 (MSS Calculation)**: 기존 데이터베이스의 모든 게시물에 대해 **MSS(Marketing Success Score)**를 계산합니다. 이때 시간의 흐름에 따른 신뢰도 하락(**Decay**)과 내부 데이터에 대한 **가중치**를 적용합니다.
3.  **유사도 클러스터링**: 3072차원 임베딩 공간에서 입력한 카피와 가장 유사한 성격의 게시물들을 묶습니다. (예: 뷰티 카피면 뷰티 게시물들만 선별)
4.  **성공 사례 추출**: 해당 클러스터에서 MSS 점수가 가장 높은 상위 5개를 추출합니다.
5.  **대조 페어 생성 (High/Low Pairs)**: 성공 사례(High)와 내용은 비슷하지만 성과가 좋지 않았던 게시물(Low)을 1:1로 매칭하여 5쌍의 대조군을 만듭니다.
6.  **동적 전략 도출 (Dynamic Arm)**: 분석 AI가 5쌍의 High/Low 차이를 분석하여 "왜 이 도메인에서는 이런 글이 터지는가?"에 대한 실시간 전술을 도출합니다.
7.  **카피 생성**: 2개의 동적 전략(Arm A/B), 1개의 하이브리드 전략(Arm C), 3개의 정적 전략을 활용하여 각 4개씩(동적+하이브리드 12개), 각 3개씩(정적 9개) **총 21개**의 카피를 생성합니다.
8.  **스코어링 및 출력**: 생성된 21개 카피의 구조와 의미를 기존 고성과 데이터 및 실시간 도출된 '금기 사항'과 비교하여 점수를 매긴 후, **상위 3개**를 최종 출력합니다.
9.  **지속 학습 (Feedback Loop)**: 실제로 업로드된 게시물은 다시 데이터베이스에 반영되어 다음 생성 시의 기준점이 됩니다.

## 🛠 2. 핵심 기술 디테일 (Deep Dive)

### 📈 MSS (Marketing Success Score) 공식
단순히 조회수가 높다고 성공한 글은 아닙니다. 시스템은 **'독자의 반응도'**를 가장 중요하게 생각합니다.
> **공식**: `(댓글조회수^2 / 본문조회수)`
> *   댓글은 단순 조회보다 훨씬 높은 적극성을 필요로 하기에 제곱 가중치를 부여합니다.

### 🕒 Decay(시간 감쇠)와 내글 가중치
*   **Decay (0.9)**: SNS 트렌드는 매일 변합니다. 날짜가 하루 지날 때마다 데이터의 신뢰도(점수)에 0.9를 곱합니다. 즉, 한 달 전의 데이터보다 오늘의 데이터가 더 강력한 영향력을 가집니다.
*   **내글 가중치 (3.0배)**: 외부에서 가져온 데이터보다 내가 직접 운영해서 성과를 낸 데이터가 더 정확한 기준이 됩니다. 따라서 내부 데이터는 외부 데이터보다 3배 더 신뢰하도록 설계되었습니다.

### 🤖 임베딩(Embedding)과 클러스터링
*   **임베딩**: 문장을 의미 단위로 쪼개어 **3072차원의 숫자 좌표**로 바꿉니다. 
*   **클러스터링**: 이 거대한 가상 공간에서 비슷한 위치에 있는 글들을 묶어주는 과정입니다. 이를 통해 "상품은 화장품인데, 전략은 자동차 리뷰인" 식의 잘못된 매칭을 사전에 차단하고, 가장 적절한 성공 공식(클러스터)만 참고하게 됩니다.

## ❓ 3. 자주 묻는 질문 (FAQ)

**Q: 데이터는 어떤 방식으로 관리되나요?**
A: 기본적으로 엑셀 표(Excel) 형태로 관리되며, 지속적인 피드백 루프를 통해 자동으로 업데이트되고 임베딩 공간에 실시간으로 반영됩니다.

**Q: 카피 한 번을 뽑을 때 API는 몇 번 호출하나요?**
A: 총 **21번**의 주요 언어 모델 호출이 발생합니다.
*   **전략 도출(1회)**: 5쌍의 대조군을 분석하여 동적 전략을 뽑을 때 1회 사용합니다.
*   **카피 생성(20회)**: 5가지 전략별로 4개씩, 총 20개의 후보군을 정밀하게 생성할 때 각각 1회씩 사용합니다.
*(임베딩 유사도 계산은 자체 로컬 서버 엔진을 사용하여 무제한으로 실시간 처리됩니다.)*

**Q: 전략당 카피 개수가 왜 정해져 있나요?**
A: 한 전략에서 너무 많은 카피를 뽑으면 형태만 바뀐 비슷한 글들만 나올 수 있습니다. 5가지의 서로 다른 관점(Arm)에서 각각의 가능성을 테스트하고 스코어러가 이를 냉정하게 채점함으로써, 가장 '확률 높은' 글을 골라내기 위함입니다.
